## 2. Flow Detail Proses

### Flow Surat Masuk dengan Disposisi

```mermaid
sequenceDiagram
    participant Staff
    participant Sistem
    participant DB_Surat
    participant DB_Disposisi
    participant DB_Log
    participant Kepala
    
    Staff->>Sistem: Input data surat masuk
    Staff->>Sistem: Upload file surat
    Sistem->>DB_Surat: Insert surat_masuk
    Sistem->>DB_Log: Insert log (status: diterima)
    DB_Surat-->>Sistem: success
    Sistem-->>Staff: Notifikasi sukses
    
    Staff->>Sistem: Pilih menu disposisi
    Sistem->>DB_Surat: Get data surat
    DB_Surat-->>Sistem: data surat
    Sistem-->>Staff: Form disposisi
    
    Staff->>Sistem: Pilih user tujuan (Kepala)
    Staff->>Sistem: Input instruksi
    Sistem->>DB_Disposisi: Insert disposisi
    Sistem->>DB_Surat: Update status = didistribusikan
    Sistem->>DB_Log: Insert log disposisi
    Sistem-->>Staff: Notifikasi disposisi terkirim
    
    Sistem-->>Kepala: Notifikasi disposisi masuk
    Kepala->>Sistem: Lihat disposisi
    Sistem->>DB_Disposisi: Get disposisi
    DB_Disposisi-->>Sistem: data disposisi
    Sistem-->>Kepala: Detail disposisi
    
    Kepala->>Sistem: Update status = diproses
    Sistem->>DB_Disposisi: Update status
    Sistem->>DB_Log: Insert log
```

### Flow Surat Keluar dengan Approval

```mermaid
sequenceDiagram
    participant Staff
    participant Sistem
    participant DB_Keluar
    participant DB_Log
    participant Kepala
    
    Staff->>Sistem: Input data surat keluar
    Staff->>Sistem: Upload file konsep
    Sistem->>DB_Keluar: Insert draft (status: draft)
    Sistem->>DB_Log: Insert log (buat draft)
    Sistem-->>Staff: Notifikasi draft tersimpan
    
    Staff->>Sistem: Submit untuk approval
    Sistem->>DB_Keluar: Update status = menunggu
    Sistem->>DB_Log: Insert log (menunggu approval)
    Sistem-->>Kepala: Notifikasi perlu approval
    
    Kepala->>Sistem: Lihat daftar approval
    Sistem->>DB_Keluar: Get surat status=menunggu
    DB_Keluar-->>Sistem: data surat
    
    Kepala->>Sistem: Approve surat
    Sistem->>DB_Keluar: Generate nomor_surat
    Sistem->>DB_Keluar: Update status = disetujui
    Sistem->>DB_Keluar: Update approved_by, approved_at
    Sistem->>DB_Log: Insert log (disetujui)
    Sistem-->>Staff: Notifikasi surat disetujui
    
    Staff->>Sistem: Upload file final
    Sistem->>DB_Keluar: Update file_path
    Sistem->>DB_Keluar: Update status = dikirim
    Sistem->>DB_Log: Insert log (dikirim)
    Sistem-->>Staff: Notifikasi surat terkirim
```

## 3. Struktur Database & Migration

### Daftar Migration (Urutan Eksekusi)

```
app/Database/Migrations/
├── 2024_01_01_000001_create_users.php
├── 2024_01_01_000002_create_surat_masuk.php
├── 2024_01_01_000003_create_surat_keluar.php
├── 2024_01_01_000004_create_disposisi.php
├── 2024_01_01_000005_create_log_aktivitas.php
└── 2024_01_01_000006_create_file_attachments.php
```

### Migration 1: Users

```php
// 2024_01_01_000001_create_users.php
{
    id (INT, PK, AI)
    username (VARCHAR 50, UNIQUE)
    password (VARCHAR 255)
    nama_lengkap (VARCHAR 100)
    email (VARCHAR 100, NULL)
    jabatan (VARCHAR 100, NULL)
    role (ENUM: admin, staff, kepala) DEFAULT 'staff'
    foto_profile (VARCHAR 255, NULL)
    created_at (DATETIME, NULL)
    updated_at (DATETIME, NULL)
    
    INDEX: idx_username, idx_role
}
```

### Migration 2: Surat Masuk

```php
// 2024_01_01_000002_create_surat_masuk.php
{
    id (INT, PK, AI)
    nomor_agenda (VARCHAR 50, UNIQUE)        // Format: IN-YYYY-XXX
    nomor_surat (VARCHAR 100)                 // Nomor dari pengirim
    tanggal_surat (DATE)                       // Tanggal surat dibuat
    tanggal_terima (DATE)                       // Tanggal diterima
    pengirim (VARCHAR 200)
    perihal (TEXT)
    lampiran (INT) DEFAULT 0
    file_path (VARCHAR 255, NULL)
    file_name (VARCHAR 255, NULL)
    file_size (INT, NULL)
    keterangan (TEXT, NULL)
    status (ENUM: diterima, didistribusikan, selesai) DEFAULT 'diterima'
    created_by (INT, NULL)                     // FK ke users.id
    created_at (DATETIME, NULL)
    updated_at (DATETIME, NULL)
    
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL
    INDEX: idx_nomor_agenda, idx_tanggal_terima, idx_status, idx_pengirim
}
```

### Migration 3: Surat Keluar

```php
// 2024_01_01_000003_create_surat_keluar.php
{
    id (INT, PK, AI)
    nomor_agenda (VARCHAR 50, UNIQUE)        // Format: OUT-YYYY-XXX
    nomor_surat (VARCHAR 100, UNIQUE)         // Nomor resmi instansi
    tanggal_surat (DATE)                       // Tanggal pembuatan
    tanggal_kirim (DATE)                       // Tanggal pengiriman
    tujuan (VARCHAR 200)                       // Tujuan surat
    perihal (TEXT)
    lampiran (INT) DEFAULT 0
    file_path (VARCHAR 255, NULL)              // Path file final
    file_name (VARCHAR 255, NULL)
    file_size (INT, NULL)
    file_konsep_path (VARCHAR 255, NULL)       // Path file draft
    keterangan (TEXT, NULL)
    status (ENUM: draft, menunggu, disetujui, dikirim) DEFAULT 'draft'
    approved_by (INT, NULL)                    // FK ke users.id
    approved_at (DATETIME, NULL)
    created_by (INT, NULL)                     // FK ke users.id
    created_at (DATETIME, NULL)
    updated_at (DATETIME, NULL)
    
    FOREIGN KEY (approved_by) REFERENCES users(id) ON DELETE SET NULL
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL
    INDEX: idx_nomor_surat, idx_tanggal_kirim, idx_status
}
```

### Migration 4: Disposisi

```php
// 2024_01_01_000004_create_disposisi.php
{
    id (INT, PK, AI)
    surat_masuk_id (INT, NOT NULL)             // FK ke surat_masuk.id
    dari_user_id (INT, NULL)                   // FK ke users.id (pengirim)
    ke_user_id (INT, NOT NULL)                 // FK ke users.id (penerima)
    instruksi (TEXT)                            // Instruksi disposisi
    batas_waktu (DATE, NULL)                    // Deadline penyelesaian
    status (ENUM: diteruskan, diproses, selesai) DEFAULT 'diteruskan'
    catatan (TEXT, NULL)                        // Catatan tambahan
    created_at (DATETIME, NULL)
    updated_at (DATETIME, NULL)
    
    FOREIGN KEY (surat_masuk_id) REFERENCES surat_masuk(id) ON DELETE CASCADE
    FOREIGN KEY (dari_user_id) REFERENCES users(id) ON DELETE SET NULL
    FOREIGN KEY (ke_user_id) REFERENCES users(id) ON DELETE CASCADE
    INDEX: idx_surat_masuk, idx_ke_user, idx_status
}
```

### Migration 5: Log Aktivitas

```php
// 2024_01_01_000005_create_log_aktivitas.php
{
    id (INT, PK, AI)
    user_id (INT, NULL)                        // FK ke users.id
    aksi (VARCHAR 50)                          // create, update, delete, view, download, disposisi, approve
    tipe_surat (ENUM: masuk, keluar)            // Jenis surat
    surat_id (INT, NOT NULL)                    // ID surat (masuk/keluar)
    detail (TEXT, NULL)                         // Detail aktivitas
    ip_address (VARCHAR 45, NULL)               // IP user
    user_agent (TEXT, NULL)                      // Browser/device info
    created_at (DATETIME, NULL)
    
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
    INDEX: idx_user, idx_tipe_surat, idx_surat_id, idx_created_at
}
```

### Migration 6: File Attachments

```php
// 2024_01_01_000006_create_file_attachments.php
{
    id (INT, PK, AI)
    surat_id (INT, NOT NULL)                    // ID surat
    tipe_surat (ENUM: masuk, keluar)            // Jenis surat
    original_name (VARCHAR 255)                  // Nama file asli
    file_path (VARCHAR 255)                      // Path file tersimpan
    file_size (INT, NULL)                        // Ukuran file (bytes)
    mime_type (VARCHAR 100, NULL)                // Tipe file
    is_konsep (BOOLEAN) DEFAULT FALSE            // True untuk file draft
    uploaded_by (INT, NULL)                      // FK ke users.id
    uploaded_at (DATETIME, NULL)
    
    FOREIGN KEY (uploaded_by) REFERENCES users(id) ON DELETE SET NULL
    INDEX: idx_surat, idx_tipe_surat
}
```

```mermaid
erDiagram
    USERS ||--o{ SURAT_MASUK : "membuat"

    USERS ||--o{ SURAT_KELUAR : "menyetujui"
    USERS ||--o{ DISPOSISI : "mengirim"
    USERS ||--o{ DISPOSISI : "menerima"
    USERS ||--o{ LOG_AKTIVITAS : "melakukan"
    USERS ||--o{ FILE_ATTACHMENTS : "mengupload"
    
    SURAT_MASUK ||--o{ DISPOSISI : "memiliki"
    SURAT_MASUK ||--o{ LOG_AKTIVITAS : "memiliki"
    SURAT_MASUK ||--o{ FILE_ATTACHMENTS : "memiliki"
    
    SURAT_KELUAR ||--o{ LOG_AKTIVITAS : "memiliki"
    SURAT_KELUAR ||--o{ FILE_ATTACHMENTS : "memiliki"
```

### Detail Kardinalitas:

| Tabel 1 | Relasi | Tabel 2 | Keterangan |
|---------|--------|---------|------------|
| users | 1 : N | surat_masuk | Satu user bisa membuat banyak surat masuk |
| users | 1 : N | surat_keluar (created_by) | Satu user bisa membuat banyak surat keluar |
| users | 1 : N | surat_keluar (approved_by) | Satu user bisa menyetujui banyak surat |
| users | 1 : N | disposisi (dari_user_id) | Satu user bisa mengirim banyak disposisi |
| users | 1 : N | disposisi (ke_user_id) | Satu user bisa menerima banyak disposisi |
| users | 1 : N | log_aktivitas | Satu user bisa memiliki banyak log |
| surat_masuk | 1 : N | disposisi | Satu surat masuk bisa banyak disposisi |
| surat_masuk | 1 : N | log_aktivitas | Satu surat masuk bisa banyak log |
| surat_keluar | 1 : N | log_aktivitas | Satu surat keluar bisa banyak log |

## Ringkasan Fitur Database

| Tabel | Fungsi | Relasi dengan |
|-------|--------|---------------|
| users | Data pengguna sistem | Semua tabel |
| surat_masuk | Data surat masuk | users, disposisi, log, file |
| surat_keluar | Data surat keluar | users, log, file |
| disposisi | Alur disposisi surat | users, surat_masuk |
| log_aktivitas | Log semua aktivitas | users, surat_masuk, surat_keluar |
| file_attachments | Manajemen file | users, surat_masuk, surat_keluar |

### Format Nomor Agenda:
- **Surat Masuk**: `IN-YYYY-NNN` (IN-2024-001)
- **Surat Keluar**: `OUT-YYYY-NNN` (OUT-2024-001)

### Status Workflow:
- **Surat Masuk**: diterima → didistribusikan → selesai
- **Surat Keluar**: draft → menunggu → disetujui → dikirim
- **Disposisi**: diteruskan → diproses → selesai
